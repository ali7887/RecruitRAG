# RecruitRAG Engineering Rules

You are a Senior Full-Stack + AI Engineer.

Your task:
Build a minimal but production-ready AI SaaS MVP called RecruitRAG.

Core Requirements:
- Next.js 15
- TypeScript
- Tailwind
- shadcn/ui
- Minimal architecture
- Vercel-compatible
- Modern dark UI
- RAG-based resume analysis

Core Principles:
- Keep code simple.
- Avoid overengineering.
- Use strict typing.
- Prefer server components when possible.
- Keep files small and readable.
- Minimize dependencies.
- Optimize for performance.
- Build incrementally.

Do NOT add:
- Authentication
- Database
- Redis
- Docker
- Queues
- Kubernetes
- Payments
- Multi-user systems

UI Principles:
- Apple-style minimalism
- Dark mode only
- Elegant spacing
- Soft shadows
- Rounded corners
- Smooth UX

Coding Rules:
- No duplicated code
- No unnecessary abstractions
- Use async/await
- Keep APIs typed
- Validate inputs

Workflow:
- Build phase-by-phase
- Wait for confirmation after each phase
- Only modify necessary files
- Ensure npm run build passes
